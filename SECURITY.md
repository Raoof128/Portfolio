# Security Policy

## Supported Scope

This policy applies to:

- `https://raoof128.github.io/Portfolio`
- Source code in this repository
- Public portfolio pages and static assets shipped from this repository

## Reporting a Vulnerability

Please report vulnerabilities privately via email:

- Contact: `raoof.r12@gmail.com`
- Subject format: `[SECURITY] <short summary>`

For encrypted reports, use the public key published at:

- `https://raoof128.github.io/Portfolio/pgp-key.txt`

## What to Include

Please include enough detail for reliable triage:

- Affected URL/path/component
- Vulnerability type and impact
- Reproduction steps
- Proof-of-concept payload or request
- Suggested mitigation (optional)

## Response Targets

- Initial acknowledgment: within 72 hours
- Triage and severity assessment: within 7 days
- Fix timeline: depends on severity and complexity

## Safe Harbor

If you act in good faith, avoid privacy violations, and do not disrupt service availability, your research will be treated as authorized testing under this policy.

Out-of-scope examples:

- Social engineering and phishing campaigns
- Denial of service testing
- Automated spam submissions
- Third-party service vulnerabilities unrelated to this codebase

## Recognition

Valid, responsibly disclosed findings may be acknowledged on:

- `https://raoof128.github.io/Portfolio/hall-of-fame`
